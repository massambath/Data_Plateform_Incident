# Data Platform Incidents (Projet en cours)

## Vision
Petit projet perso pour rester sharp sur la data et l’ops.  
L’objectif : construire une mini plateforme qui collecte, stocke et expose des données d’incidents, du raw data jusqu’aux dashboards.

## Stack
- Python (ingestion + API)
- PostgreSQL (stockage + SQL views)
- Docker (containerisation)
- Kubernetes (orchestration)
- Grafana (dashboards)

## Roadmap
1. Définir le schéma SQL
2. Créer le service d’ingestion Python
3. Créer API analytics FastAPI
4. Dockeriser les services
5. Déploiement via Docker Compose et Kubernetes
6. Dashboards Grafana pour visualisation
7. Tests unitaires et observabilité

> Chaque étape est indépendante et permet de progresser semaine après semaine.
